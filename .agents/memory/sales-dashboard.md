# Sales Performance Dashboard

Standalone artifact `sales-dashboard` (slug/previewPath `/sales-dashboard/`), separate from the task-manager. It is the original 4266-line standalone HTML (`attached_assets/Infinity_Sales_Performance_Dashboard_*.html`) hosted as-is (Vite serves `index.html`; the react-vite `src/` is dead/unused — do NOT reintroduce a React entry).

## One-way data flow (Sheet → dashboard)
- Server route `GET /api/sales-dashboard/data` (PUBLIC — registered before `requireAuth` in api-server) returns the full payload. Root-relative `/api/...` is correct (shared proxy routes /api globally).
- Payload is built by `@workspace/sales-dashboard-data` `buildPayload(rawTables)`. Raw tables come from the Google Sheet when `GOOGLE_SHEET_ID` env is set (read via `sheetSource.ts` using the `google-sheet` connector runtime token), otherwise from bundled `seedRawTables` (current numbers). Sheet read failure → HTTP 502 (no silent fallback); client shows an error overlay with Retry.

## The 9 render objects & what is RAW vs DERIVED
Client HTML consumes 9 globals: FY1, FY2, SP_FY1, SP_FY2, HC_BRANCH_DATA, HC_CENTER_TOTALS, MD_CENTER_DATA, MD_CONV, ATL_RAW(918 agent rows). Surgery converted their `const … = {literal}` to `let X = null` and a fetch bootstrap assigns them then runs switchFY(1)/atlApplyFilters/initSalesPage/initHCPage/initMandaysPage.
- **Aggregate rows are stored RAW in the Sheet, NOT derived** — the source's Overall/ALL rows are internally inconsistent with column sums (FY2.Overall.proj off by 1 in 2 months; Mandays ALL.leads independent of center sums; per-center Mandays conv ≠ sales/leads). Deriving them would visibly change the mandays conversion line. So Sales tab has an `Overall` series row per FY, and Mandays tab has an `ALL` center row. HC_CenterTotals `ALL` is also an independent raw row.
- **Only truly computed value in buildPayload:** HC_CENTER_TOTALS `ach` = (actual/proj*100).toFixed(1)+'%'. All other "derived" scalars (gaps, totals, rankings, best/weak) are computed client-side at render, untouched.
- SP_FY1/SP_FY2 are derived from FY1/FY2 (SP.ALL=FY.Overall; SP_FY2.Airoli=FY2.NaviMumbai; colors/centers are structural constants in the builder). MD_CONV = per-center MD_CENTER_DATA.conv.

## Live editable sheet (July 2026)
A real user-editable Google Sheet now exists in the connected google-sheet account (5 tabs matching the layout above), created from the full seed data as starter/example rows; `GOOGLE_SHEET_ID` (shared env) points to it, so dev + prod read live from it. **Gotcha (July 2026): TWO same-named spreadsheets ("Infinity Sales Performance Dashboard — Data") exist in that Drive — the user edits one while GOOGLE_SHEET_ID pointed at the stale duplicate, so edits "didn't show". If the user reports Sheet edits not appearing, FIRST compare the sheet read by the env id against a Drive search by modifiedTime (duplicate-copy check) before touching code. Env now points at the copy the user actually edits.** Client polls the data route every 5s (silent, state-preserving) and the route has a 4s in-memory micro-cache so many open tabs share one Sheet read. ## Tolerant per-row parsing (July 2026)
- A mistyped cell no longer 502s the dashboard: numeric coercers first strip currency symbols/commas/% ("₹1,200"→1200, "85%"→85), and each row is mapped in its own try/catch — a still-unparseable row is SKIPPED and recorded (tab + sheet row number, header-aware +2 offset).
- The data payload carries a `skippedRows: string[]` field (also warn-logged server-side) so typos are diagnosable; the static HTML currently ignores it.
- Real breakage still errors: if a REQUIRED tab (Sales/Mandays/Agents) yields zero usable rows the read throws (with first few skip reasons) → 502. Don't "fix" that into a silent empty render.

## Validation invariant
`buildPayload(seedRawTables)` must deep-equal the objects extracted from the original HTML (all 9). The seed was generated by decomposing those extracted objects, and the round-trip was verified byte-exact.

## Sales Report page (rp* functions)
- A "Sales Report" page (nav → `showPage('report')` → `initReportPage()`) replicates the Power BI report: 4 KPIs, Year/Month/Center tile filters, 6 Chart.js charts. All logic lives inline in `index.html` prefixed `rp*` (own `rpFilters`/`rpCharts` state; no clash with the rest).
- **Time-domain gotcha:** `ATL_RAW` has sales rows for MORE months than `MD_CENTER_DATA` has mandays (mandays arrays are only the first 12 FY months, `ATL_MONTHS_ORDER.slice(0,12)`). A conversion ratio (`sales/mandays`) MUST restrict its sales numerator to the same month-set that has mandays (`rpMandayMonthKeys`), or Convn is inflated/inconsistent for filters hitting mandays-less months (e.g. Year=2026, Apr/May). Total Sale / Total DC / center+agent+TL+month charts intentionally still show ALL filtered sales — only the Convn KPI and the Man Days Vs Conv line are windowed.
- **Why:** numerator and denominator on different time domains silently produce a wrong ratio that still "looks fine" on screen.
- Report centers are data-driven from `MD_CENTER_DATA` keys (minus `ALL`) via `rpCenters()`, colored by `rpColor()` (known-center map + fallback palette), so a new center in the Sheet won't be silently dropped.

## Fit-to-screen zoom (July 2026)
- `.main` has `zoom: 0.7` (reset to 1 via `@media (max-width:1100px)`) so the whole dashboard renders like ~67% browser zoom without the user manually zooming out. User asked for this baked-in default.
- **Gotcha (crispness):** CSS `zoom` downscales the Chart.js `<canvas>` bitmaps, which blurs the number datalabels. Fix: `Chart.defaults.devicePixelRatio = Math.max(2, (window.devicePixelRatio||1)/0.7)` set right after `Chart.register(ChartDataLabels)` — renders canvases at higher res so they stay crisp after the zoom downscale. If the zoom value changes, keep this DPR in step.
- **Gotcha (black numbers):** ChartDataLabels is registered globally, so EVERY chart shows labels by default — and any chart that doesn't set its own `datalabels.color` inherits the plugin default `#666` (dark gray) which reads as black on the dark bg (e.g. ATL page Top 10 Agents / TL Sales / TL Leads bar charts have no per-chart datalabel config). Fix: set `Chart.defaults.plugins.datalabels.color = '#dbe4fb'` (+ default font) right after register. Any new chart without an explicit label color is now readable automatically.
## Click-to-filter / cross-filter (Power BI-style, July 2026)
- User wanted "Power BI system on all charts" = clicking a chart element filters the page (they explicitly picked click-to-filter, not tooltips/slicers).
- Implemented as ONE global `Chart.defaults.onClick` + `onHover` (right after the datalabel defaults ~line 1223). onClick → `sdCrossFilter(chart, {index,datasetIndex})` dispatcher keyed by `chart.canvas.id`, which just calls the EXISTING per-page filter setters with toggle-to-ALL (re-click clears): Overview pie→`applyCenter`; sp-center-fy1/fy2→`spSetCenter`; md-conv-chart→`mdSetCenter` (uses DATASET label = center when multi-line ALL mode); atlChartCenterSales→`atlTileFilter('center')`; atlChartTLSales/Leads→`atlTileFilter('tl')`; rpChartMandays/rpChartCenter→`rpSetFilter('center')`; rpChartMonth→`rpSetFilter('month')`. Month/time-series charts on pages with no month filter are intentionally inert (no new filter dimension built).
- **Gotcha (MUST defer):** applying a filter destroys+recreates the charts; doing that synchronously inside Chart.js's own click dispatch throws an uncatchable async "unknown runtime error" (Vite overlay) — intermittent (center seemed fine, month failed). Fix: `setTimeout(fn,0)` capturing index/datasetIndex first. Keep any future chart-click handlers deferred.
- **Gotcha (bar hit target):** clicking just above a thin bar returns no element; added nearest-index fallback `chart.getElementsAtEventForMode(evt,'index',{intersect:false},true)` so a click anywhere in the column selects it (Power BI feel). Direct hit still preferred. The fallback is REJECTED for line datasets (`getDatasetMeta(dsIdx).type==='line'`) — on multi-line md-conv-chart an index-only match would silently pick the first line = wrong center; there a direct hit on the point is required.
- **Gotcha (label≠filter value):** rpChartMonth labels are `Apr-25` but the month filter/tiles use short `Apr` → convert with `rpParseMonth(label).mon`. TL charts label bars with first-name only (`t.tl.split(' ')[0]`) → full TL names stashed on `chart.$sdTL` at creation for `atlTileFilter('tl', fullName)`. `sdNormCenter()` maps `Navi Mumbai`→`Airoli`.

## Center color palette (canonical, July 2026)
- User-mandated per-CENTER colors (from a Power BI CentreColor SWITCH measure): Malad `#00FFFF` (aqua), Thane `#ADFF2F` (green-yellow), Pune `#FF6347` (tomato), Airoli & Navi Mumbai `#FFD700` (gold), Noida `#1E90FF`, NP/Nariman Point `#D6DBDF`. Overall/ALL stays `#3d7fff`.
- **Gotcha (overloaded hexes):** the OLD center hexes (`#ff4d6d`,`#00d4ff`,`#a78bfa`,`#00e676`,`#ff9f43`) are ALSO the semantic CSS vars (--red/--cyan/--purple/--green/--orange) and reused for gaps/attrition/ach%/FY-series — so a global find/replace corrupts unrelated UI. Recolor ONLY center-scoped spots: the JS center maps (OV_BRANCH_COLORS, ATL_CENTER_COLORS, HC_COLORS, MD_COLORS, RP_CENTER_COLORS + RP_FALLBACK first 4, brColors×2, cols×2, per-center data `color`/`vc`/`doughnutColors`/`convLines` in the FY/center data blocks) and the center tiles/buttons/dots (cf-btn/sp-tile/md-tile/atl-tile/hc-center-btn/hc-br-btn + sidebar Branch Sales dots). The NEW palette values do NOT collide with any existing semantic hex, so they're safe once set. FY-series colors on the Overview compare charts (green=FY1, orange/purple=FY2) are NOT centers — leave them.
- ATL page per-TL bar colors (`TL_COLORS`, rainbow) left as-is (not center-based) — only TL tile BORDERS follow their center color.

- **Gotcha (size):** DPR crispness alone was NOT enough — at zoom 0.7 the original 8–9px datalabels rendered ~6px effective and the user said numbers "still don't show properly." The real fix was bumping datalabel FONT SIZES: all `datalabels:{display:true...font:{size:N}}` were increased +3 (now 11/12/13) and the DL_BAR default color brightened `#8896b8`→`#c6d3f2`. Axis/legend fonts left as-is. Rule: any global shrink (zoom) must be paired with larger number-label fonts or the numbers become unreadable even if crisp.

- **Missing-branch crash guard (July 2026):** the user editing the Sheet can silently drop a whole branch section (e.g. FY2 Navi Mumbai/Airoli) while `centers` lists still name it — render code then threw `undefined.proj` INSIDE the boot `setTimeout` (uncaught → "Loading dashboard…" spinner frozen forever, dev AND prod). `sdNormalize(data)` (called in both sdBootstrap and sdSilentRefresh, right after the SD_KEYS check) backfills every hardcoded branch/center key across FY1/FY2/HC_BRANCH_DATA/MD_CENTER_DATA (incl. `leads`)/MD_CONV/SP_FY1/SP_FY2 with zero/null arrays. FY1 actuals fill 0 (render has no null guards), FY2 actuals fill null (YTD "—" semantics). If a new hardcoded branch key or a new field on these objects is added to render code, extend sdNormalize or the freeze can come back.
- **Series-name canonicalization (July 2026):** the Sheet's Sales tab series column is hand-typed — the user wrote "Navi Mumbai" (space) while the payload builder reads exact key `NaviMumbai`, silently dropping the whole branch. sheetSource now canonicalizes known series (trim + strip spaces + case-insensitive → Overall/Pune/Malad/Thane/Airoli/NaviMumbai) before buildPayload. If a new branch is ever added, add it to SERIES_CANON too or hand-typed variants will vanish silently.
- **Axis scales hidden (July 2026, user request):** numeric y/y2 tick columns are hidden GLOBALLY via `Chart.defaults.scales.linear.ticks.display=false` (x/category month labels stay). Values live on datalabels instead. Don't re-add per-chart tick colors expecting them to show; to unhide one chart set `ticks:{display:true}` explicitly.
- **Sheet date cells → Excel serials (July 2026):** the API reads with `valueRenderOption=UNFORMATTED_VALUE`, so if the user reformats a Month/DOJ cell into a real date it arrives as an Excel serial (e.g. 45748) and broke the Sales Report Year filter. sheetSource.ts now normalizes serials (epoch 1899-12-30, UTC, floor, plausible-range guard 36526–73050) back to "Apr-25" / "01-Apr-25" for ALL month/date columns. Don't remove this — the Sheet is hand-edited and formats will drift again.

## Write-back editing (dashboard → Sheet, July 2026)
- The dashboard is no longer strictly one-way: authorized staff edit Sales (target/actual) and Mandays (mandays/sales/conv/leads) in an in-page modal (✏️ Edit Data button in the sidebar footer), which POSTs to `/api/sales-dashboard/update` and writes straight back to the SAME Google Sheet. The google-sheet connector already has read-write `spreadsheets` scope (not just readonly), so writes work with the same runtime token as reads.
- **Auth model:** the dashboard is a PUBLIC static page but the api-server session cookie (`iss_uid`, from the task app) rides along on same-origin `/api` calls. `/session` (public, optional cookie via `userFromCookie` — never 401s anon) returns `{canEdit}` and only then shows the edit button. `/update` is hard-gated `requireAuth + requireBossOrMis` (Boss/MIS/Director) — hiding the button is convenience, the server gate is the boundary. Both `/data` and `/session` stay public (registered before the global requireAuth via the salesDashboard router).
- **Write mechanism = cell-level, match-by-key (sheetSource.writeSheetCells):** each edit is resolved to a specific A1 cell by reading the tab and finding the row whose key columns match (canonicalized the SAME way as the read path — that's why SERIES_CANON/canonMonth/canonSeries were hoisted to module scope). Only the numeric cell (D/E for Sales, C–F for Mandays) is written via `values:batchUpdate` RAW; the month/key cells are never rewritten (no format drift). **Concurrency = last-write-wins per cell** (different cells don't clobber; same cell overwrites). If ANY edit can't match an existing row the whole request fails BEFORE writing (no half-apply) → 400 "No Sales/Mandays row for …"; a real Sheet outage → 502. A successful write clears the route's 4s micro-cache so the editor's own poll shows it immediately (client also force-refreshes via sdSilentRefreshNow).
- **Blank Actual = null clears the cell** (YTD "not yet"); blank on other numeric fields = "leave unchanged" (field omitted). Editing needs a real Sheet — with only the bundled seed `/update` returns 409 (nowhere to persist).
